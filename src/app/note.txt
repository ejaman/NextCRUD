1. file base 라우팅
: page는 route마다 고유한 UI로 page.js 파일을 통해 정의할 수 있음

1-1. static routing: (정적)라우팅 위치 고정

기존 12에선 페이지 안에 파일만 만들면 가능
13에선 폴더 안에 page.tsx를 만들어야 함
부모 라우팅 내에서 재사용 가능한 컴포넌트를 지정해 주기 위해서
로딩 상태, 에러 상태 등을 처리해 주기 위해 폴더에서 처리 해당 라우트 안에서 처리하기 위해
-> 해당 경로별 로딩바, 에러상태, 재사용 가능한 레이아웃 골격 등을 지정해서 사용 가능
-> 더 유연한 웹 사이트 만들기 가능

1-2. Dynamic routing: 정해진 경로x, 동적으로 접근 가능
-> 빌드를 할 떄 페이지를 미리 만들어 둘 수 없었음

골격은 미리 pre-rendering 되어 있지만 
동적인 파트는 동적으로 생성해서 SSR처럼 만들어져서 클라이언트에게 보내짐

그래서 우리가 원하는 경로의 한하여 페이지를 미리 만들어 두고 싶다면
미리 만들어 두고 싶은 페이지 명시 generateStaticParams
